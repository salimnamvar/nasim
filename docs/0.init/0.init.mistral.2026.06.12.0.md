## **1. Architecture Overview**
### **Client-Server Model**
- **Client (CLI):**
  Runs on the user’s local machine.
  Handles user input, terminal I/O, and displays responses.
  Communicates with the **Server** via HTTP/HTTPS or WebSockets.
  **Responsibilities:**
  - Parse user commands (e.g., code requests, file edits).
  - Stream real-time output to the terminal.
  - Manage local state (history, context, session files).

- **Server:**
  Hosts the LLM (via **Ollama** or **LM Studio**).
  Exposes a REST/WebSocket API compatible with the **Anthropic Messages format**.
  **Responsibilities:**
  - Process LLM requests.
  - Manage tool use (filesystem, shell commands) via the **Bridge** (see Section 5).
  - Handle authentication (API keys, environment variables).

---
### **Communication Flow**
```
User → CLI (Client) → HTTP/WebSocket → Server (LLM + Bridge) → Tool Execution → Response → CLI → User
```

## **2. API Compatibility: Anthropic Messages Format**
### **Why Anthropic Messages Format?**
- Ensures compatibility with existing tools (e.g., Claude Code, SDKs).
- Standardizes input/output for messages, tool use, and streaming.

### **Implementation**
- **Request Format:**
  Use the [Anthropic Messages API schema](https://docs.anthropic.com/en/api/messages):
  ```json
  {
    "model": "local-llm",
    "max_tokens": 1024,
    "messages": [
      {"role": "user", "content": "Write a Python function to sort a list."}
    ],
    "tools": [{"name": "shell", "description": "Execute shell commands."}]
  }
  ```
- **Response Format:**
  Mirror Anthropic’s response structure, including `content`, `tool_use`, and `stop_reason`.

- **Streaming:**
  Support **Server-Sent Events (SSE)** or WebSockets for real-time token streaming.

---
### **API Endpoints**
| Endpoint          | Method | Description                          |
|-------------------|--------|--------------------------------------|
| `/api/messages`   | POST   | Send a message, receive LLM response.|
| `/api/tools/use`  | POST   | Execute a tool (e.g., shell command).|

## **3. Environment Redirection**
### **Toggle Between Official Anthropic API and Local Server**
Use environment variables to dynamically switch the target API:

```bash
# For official Anthropic API
export ANTHROPIC_BASE_URL="https://api.anthropic.com"
export ANTHROPIC_API_KEY="your-anthropic-key"

# For local server (default: http://localhost:8080)
export ANTHROPIC_BASE_URL="http://localhost:8080"
export ANTHROPIC_API_KEY="dummy-key"  # Optional: Disable auth for local use
```

### **Implementation in Client**
- **Priority Order:**
  1. Check `ANTHROPIC_BASE_URL`. If set, use it.
  2. Fall back to `http://localhost:8080` (default for local server).
- **Code Snippet (Python Example):**
  ```python
  import os
  import requests

  base_url = os.getenv("ANTHROPIC_BASE_URL", "http://localhost:8080")
  api_key = os.getenv("ANTHROPIC_API_KEY", "local-key")

  headers = {"x-api-key": api_key, "Content-Type": "application/json"}
  response = requests.post(f"{base_url}/api/messages", json=payload, headers=headers, stream=True)
  ```

---
### **Docker/Proxy Setup (Optional)**

  ```nginx
  server {
    listen 8080;

      proxy_pass $ANTHROPIC_BASE_URL;
  ```

## **4. Local Knowledge Repo: Filesystem Structure**
### **Directory Layout**
Inspired by `~/.claude/` but optimized for performance:
```
~/.ai-assistant/
├── sessions/          # Session history (JSON/JSONL)
│   ├── session_123.json
│   └── session_456.json
├── context/           # Project-specific context (e.g., codebase embeddings)
│   ├── project_a/
│   │   ├── embeddings.pkl
│   │   └── metadata.json
│   └── project_b/
├── tools/             # Tool configurations (e.g., allowed shell commands)
│   └── allowed_commands.json
├── cache/             # LLM response cache (SQLite or JSON)
│   └── responses.db
└── config.json        # Global settings (e.g., default model, API timeout)
```

### **Performance Optimizations**
- **Session History:**
  - Store as **JSONL** (append-only) for easy streaming and parsing.
  - Index by `session_id` and `timestamp` for fast retrieval.
- **Context/Embeddings:**
  - Use **SQLite** or **FAISS** for vector similarity search.
  - Cache embeddings for frequently accessed files.
- **File Watching:**
  - Use **`watchdog`** (Python) or **`chokidar`** (Node.js) to auto-update context when files change.

## **5. Tool Use & The Bridge**
### **What is the Bridge?**
A **translation proxy** that:
1. Intercepts tool calls from the LLM (e.g., `{"name": "shell", "arguments": {"command": "ls -la"}}`).
2. Executes them **securely** in a sandboxed environment.
3. Returns the output to the LLM for further processing.

### **Implementation**
#### **A. Tool Definitions**
Define tools in a schema (similar to Anthropic’s `tools` parameter):
```json
{
  "tools": [
    {
      "name": "shell",
      "description": "Execute a shell command. Use with caution.",
      "input_schema": {
        "type": "object",
        "properties": {
          "command": {"type": "string"}
        }
      }
    },
    {
      "name": "read_file",
      "description": "Read a file's contents.",
      "input_schema": {
        "type": "object",
        "properties": {
          "path": {"type": "string"}
        }
      }
    }
  ]
}
```

#### **B. Bridge Server (Python Example)**
```python
from fastapi import FastAPI, HTTPException
import subprocess
import os

app = FastAPI()

ALLOWED_COMMANDS = ["ls", "cat", "grep"]  # Whitelist for security

@app.post("/api/tools/shell")
def execute_shell(command: dict):
    cmd = command["command"].split()
    if cmd[0] not in ALLOWED_COMMANDS:
        raise HTTPException(status_code=403, detail="Command not allowed")
    try:
        result = subprocess.run(cmd, capture_output=True, text=True, cwd=os.getcwd())
        return {"output": result.stdout, "error": result.stderr}
    except Exception as e:
        raise HTTPException(status_code=500, detail=str(e))
```

#### **C. Security Considerations**
- **Sandboxing:**
  - Use **Docker containers** or **`bubblewrap`** to isolate tool execution.
  - Restrict filesystem access (e.g., chroot, user permissions).
- **Input Validation:**
  - Whitelist allowed commands/files.
  - Sanitize inputs (e.g., prevent `; rm -rf /`).
- **Timeouts:**
  - Kill long-running processes (e.g., `timeout=10s`).

---
### **D. Tool Use Flow**
1. LLM generates a tool call (e.g., `{"name": "shell", "arguments": {"command": "ls"}}`).
2. Bridge validates the call and executes it.
3. Bridge returns the output to the LLM.
4. LLM incorporates the output into its next response.

---
### **E. Advanced: File Edits**
- Use **`difflib`** (Python) or **`jscpd`** (Node.js) to generate and apply patches.
- Example:
  ```python
  def apply_edit(file_path: str, old_text: str, new_text: str):
      with open(file_path, "r") as f:
          content = f.read()
      if old_text not in content:
          raise ValueError("Old text not found")
      content = content.replace(old_text, new_text)
      with open(file_path, "w") as f:
          f.write(content)
  ```

## **6. Technical Stack Recommendations**
### **Language & Framework**
| Component          | Recommended Stack                          | Rationale                                                                 |
|--------------------|--------------------------------------------|---------------------------------------------------------------------------|
| **CLI Client**     | Python (Click/Typer) or Rust (Clap)        | Python: Rich ecosystem, easy async. Rust: Performance, safety.          |
| **Server**         | Python (FastAPI) or Node.js (Express)      | FastAPI: Async, OpenAPI docs. Express: JS ecosystem.                     |
| **Bridge**         | Python (subprocess + Pydantic)             | Easy to integrate with LLM tool calls.                                  |
| **Terminal I/O**   | Python (`rich` library) or Node.js (`ink`)  | `rich`: Syntax highlighting, live display. `ink`: React-like components. |
| **Shell Execution**| Python (`subprocess`) or Node.js (`child_process`) | Sandbox with `bubblewrap` or Docker.               |
| **Local LLM**      | Ollama or LM Studio                        | Ollama: Lightweight, CLI-friendly. LM Studio: GUI, model management.     |

---
### **Libraries for Real-Time Terminal Output**
| Task               | Python Library       | Node.js Library       |
|--------------------|----------------------|------------------------|
| Streaming          | `aiohttp`            | `axios` + `EventSource`|
| Terminal UI        | `rich`, `textual`    | `ink`, `blessed`      |
| File Watching      | `watchdog`           | `chokidar`            |
| Process Management | `asyncio` + `subprocess` | `child_process`   |

---
### **Example: FastAPI Server (Python)**
```python
from fastapi import FastAPI, Request
from fastapi.responses import StreamingResponse
import ollama

app = FastAPI()

@app.post("/api/messages")
async def generate_message(request: Request):
    payload = await request.json()
    stream = ollama.chat(
        model=payload["model"],
        messages=payload["messages"],
        stream=True,
    )
    return StreamingResponse(stream, media_type="text/event-stream")
```

---
### **Example: CLI Client (Python with `rich`)**
```python
from rich.console import Console
from rich.live import Live
import httpx

console = Console()
base_url = os.getenv("ANTHROPIC_BASE_URL", "http://localhost:8080")

async def stream_response(messages):
    async with httpx.AsyncClient() as client:
        async with client.stream("POST", f"{base_url}/api/messages", json={"messages": messages}) as response:
            async for line in response.aiter_lines():
                if line.startswith("data:"):
                    data = json.loads(line[5:])
                    console.print(data["content"], end="")
```

## **9. Local State Management**
### **Session Handling**
- **Save Sessions:**
  ```python
  def save_session(session_id: str, messages: list):
      os.makedirs("~/.ai-assistant/sessions", exist_ok=True)
      with open(f"~/.ai-assistant/sessions/{session_id}.json", "w") as f:
          json.dump({"id": session_id, "messages": messages}, f)
  ```
- **Load Sessions:**
  ```python
  def load_session(session_id: str):
      with open(f"~/.ai-assistant/sessions/{session_id}.json", "r") as f:
          return json.load(f)
  ```

### **Context Caching**
- Use **SQLite** for structured data (e.g., file embeddings):

## **12. Example: Full Workflow**
1. **User runs:**
   ```bash
   ai-assistant --prompt "Refactor the code in src/app.py."
   ```
2. **CLI:**
   - Loads `~/.ai-assistant/sessions/last_session.json`.
   - Sends prompt to `http://localhost:8080/api/messages`.
3. **Server:**
   - Processes request with Ollama.
   - LLM responds with a tool call: `{"name": "read_file", "arguments": {"path": "src/app.py"}}`.
4. **Bridge:**
   - Reads `src/app.py` and returns content.
5. **LLM:**
   - Generates refactored code and a tool call: `{"name": "write_file", "arguments": {"path": "src/app.py", "content": "..."}}`.
6. **Bridge:**
   - Writes the new content to `src/app.py`.
7. **CLI:**
   - Displays the refactored code and saves the session.
