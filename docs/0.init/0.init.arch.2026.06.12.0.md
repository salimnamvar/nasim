Here is a comprehensive blueprint to get Claude Code working with your own local LLM, running entirely on your `black` server and accessible from your `salim-hp` laptop.

### 🤔 How It Works: The High-Level Plan

At a high level, the process involves three components that work together:
*   **Server (`black`)**: This is the powerhouse, running the LLM and an API that speaks the **Anthropic Messages format**, which is the "language" Claude Code understands natively.
*   **Client (`salim-hp`)**: This is your `salim-hp` laptop, running the Claude Code CLI. It sends requests to the server's API instead of to Anthropic.
*   **The Bridge**: The key to making Claude Code work with any local model is directing its API calls to your server. This is done by setting a few environment variables on the client machine.

This blueprint will guide you through setting up each of these parts to create a powerful, private, and cost-free AI coding assistant.

### 🖥️ Step 1: Setting Up the LLM Server (`black`)

This section covers installing Ollama, which is the recommended starting point for its ease of use and robust feature set. Steps for an alternative setup using LM Studio are provided afterward.

#### **Option A: Using Ollama (Recommended)**

**1. Install Ollama**
Use the official automated install script:
```bash
curl -fsSL https://ollama.com/install.sh | sh
```
This will install Ollama as a systemd service, meaning it will automatically start in the background every time your server boots up. After installation, the Ollama service will be running, and you can verify this with `sudo systemctl status ollama`.

**2. Configure Ollama for Network Access**
By default, Ollama only listens for connections from the same machine (`localhost`). To allow your laptop to connect, you need to set an environment variable. Follow the steps below to configure this via a systemd override, which ensures the setting persists across updates.

First, edit the systemd configuration:
```bash
sudo systemctl edit ollama
```
This will open a blank override file. Add the following lines:
```ini
[Service]
Environment="OLLAMA_HOST=0.0.0.0:11434"
```
Save the file and exit the editor. This instructs the Ollama service to listen on all network interfaces on port `11434`. Finally, restart the Ollama service to apply the change:
```bash
sudo systemctl restart ollama
```

**3. Allow the Port Through the Firewall**
If your server has a firewall (like the common `ufw`), you must allow incoming connections on port `11434`:
```bash
sudo ufw allow 11434/tcp
```
If you are using a different firewall, such as `firewalld`, you'll need to use its respective command to open the port.

**4. Download a Code-Optimized LLM**
SSH into your `black` server and pull a powerful, coding-specific model. `qwen3-coder` is an excellent choice:
```bash
ollama pull qwen3-coder
```
Ollama supports many open-source models. You can view a full list on [Ollama's library](https://ollama.com/library).

**5. Test Your Server**
Finally, verify that the server is accessible from your laptop (`salim-hp`). First, find your server's local IP address by running `ip a` or `hostname -I` on `black`. Then, on `salim-hp`, run this command in a terminal:
```bash
curl http://<BLACK_SERVER_IP>:11434
```
If the server is set up correctly, you should see the Ollama API's `Ollama is running` message.

#### **Option B: Using LM Studio (Alternative)**

If you prefer a more visual approach, LM Studio is an excellent alternative.

**1. Download and Run on `black`**
Download the Linux (or Windows) version for your server from the [LM Studio website](https://lmstudio.ai/). After installation, launch it and search for and download a model like `Qwen3-Coder`.

**2. Configure the Server**
In the LM Studio interface, go to the **Developer** page. Under the server section, make sure you toggle on the **"Serve on Local Network"** setting. This ensures the server is accessible from other devices. Note the port number (the default is `1234`).

**3. Test Your Server**
To test, run the following command on your laptop:
```bash
curl http://<BLACK_SERVER_IP>:1234/v1/models
```
If successful, you'll see a JSON list of models. This confirms your server is running and correctly exposing the Anthropic API format.

### 💻 Step 2: Setting Up the Client (`salim-hp`)

Now, configure your `salim-hp` laptop to use the local server as its brain.

**1. Install Claude Code**
Install Claude Code globally via npm using the following terminal command:
```bash
npm install -g @anthropic-ai/claude-code
```
**Please note:** The official installation documentation recommends the use of the install script on the [Claude Code setup page](https://code.claude.com/docs/en/setup) for the most up-to-date and reliable method. However, the `npm` command is generally the quickest way for developers already familiar with the Node.js ecosystem.

**2. Configure Environment Variables**
This is the most crucial step for redirection. You can set these variables for your entire user session by adding them to your shell's configuration file (e.g., `~/.bashrc`, `~/.zshrc`), or for a single session directly in the terminal.

**🚨 Important:** The environment variable `ANTHROPIC_API_KEY` must be set to a **non-empty placeholder string** (like `"local"`). If this is unset or empty, Claude Code will attempt to authenticate in other ways (e.g., browser login) and fail to connect.

```bash
# The base URL of your Ollama server
export ANTHROPIC_BASE_URL="http://<BLACK_SERVER_IP>:11434"

# A mandatory placeholder API key
export ANTHROPIC_API_KEY="local"

# Map Claude Code's requested model names to your local model
export ANTHROPIC_DEFAULT_SONNET_MODEL="qwen3-coder"
export ANTHROPIC_DEFAULT_OPUS_MODEL="qwen3-coder"
export ANTHROPIC_DEFAULT_HAIKU_MODEL="qwen3-coder"
```
Set `ANTHROPIC_BASE_URL` to your local server address to redirect all API calls.

**3. (Optional) Enable Model Discovery**
To make the `/model` command in Claude Code list the models available on your server, use this environment variable:
```bash
export CLAUDE_CODE_ENABLE_GATEWAY_MODEL_DISCOVERY=1
```
When `ANTHROPIC_BASE_URL` points to a gateway that exposes the Anthropic Messages format, Claude Code can query the gateway's `/v1/models` endpoint at startup and add the returned models to the `/model` picker.

**4. (For Ollama Only) Ensure the Server is Running**
If you are using Ollama, you may need to ensure the server is running. While you should have set `OLLAMA_HOST` earlier, it's a good practice to start the server on the server machine before launching the client. Run the following command on your `black` server machine's terminal (not on `salim-hp`):
```bash
ollama serve
```

---

### 🚀 Step 3: Running Claude Code

After setting the environment variables, run Claude Code from the same terminal session on `salim-hp`:
```bash
claude
```

If the setup is successful, Claude Code will start without asking for an API key, directly connected to your local model.

### 🔌 Going Beyond: The Full Compatibility Bridge

While Ollama's native Anthropic API endpoint is great, it may not support all of Claude Code's advanced features (like sophisticated tool use). For the most complete and reliable experience, using a translation proxy is the safest route.

**1. Running a Proxy Server on `black`**
The `claude-proxy` by `@beeplovsharma` is a simple, single-command solution for this.

Install and run it on `black`:
```bash
npm install -g claude-proxy
```
Then, start the proxy server:
```bash
claude-proxy start
```
This command will launch the proxy, which automatically converts Claude Code's requests to the standard format required by your local model.

**2. Configuring the Client**
On your `salim-hp` laptop, point Claude Code to your proxy server. No other configuration is needed:
```bash
export ANTHROPIC_BASE_URL="http://<BLACK_SERVER_IP>:4000"
claude
```

### ❓ Troubleshooting

If you encounter issues, here are the most common pitfalls:

*   **Connection Refused**: The most common issue. Ensure the server's firewall is open (`sudo ufw allow 11434/tcp`), the Ollama server is bound to `0.0.0.0`, and you are using the correct local IP address on the client.
*   **Authentication Errors**: Always set `ANTHROPIC_API_KEY` to a non-empty placeholder string, like `"local"`, on the client machine. Leaving it empty or unset will cause Claude Code to look for a real Anthropic key.
*   **Model Not Found Errors**: If you see a "model not found" error, it means the model name being requested isn't available. Use the `ANTHROPIC_DEFAULT_*` environment variables to map the requested model names (like `claude-sonnet-4-20250514`) to the name of your local model (`qwen3-coder`).
*   **Slow Performance**: Some models, especially larger ones, can be slow on CPU-only servers. Using a GPU or selecting a smaller model (e.g., `qwen3-coder:1.7b`) can significantly improve speed. You can check the running models on your server with `ollama ps`.
