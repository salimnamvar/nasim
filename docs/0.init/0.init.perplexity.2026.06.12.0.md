# **Technical Blueprint: Terminal-Native Local AI Coding Assistant**

## **Executive Summary**

Build a Client-Server AI coding assistant where a Rust/Python CLI client communicates with a local LLM server (Ollama/LM Studio) speaking the Anthropic Messages format natively, with environment variable redirection, a local knowledge repo, and an MCP-based tool bridge for filesystem/shell operations.

***

## **1. Architecture Overview: Client-Server Model**

### **High-Level Architecture**

```
┌─────────────────────┐         ┌───────────────────────┐
│  CLIENT (CLI)       │         │  SERVER (LLM Backend) │
│  ─────────────────  │         │  ───────────────────  │
│  • Rust/Python CLI  │◄───────►│  • Ollama/LM Studio   │
│  • Anthropic API    │  HTTP   │  • Anthropic Messages │
│  • Tool Executor    │  SSE    │  • Model Router       │
│  • Local Knowledge  │         │  • MCP Bridge         │
└─────────────────────┘         └───────────────────────┘
```

### **Component Breakdown**

| Component | Responsibility | Recommended Tech |
|-----------|---------------|------------------|
| **CLI Client** | Terminal UI, prompt parsing, tool execution | **Rust** (with [Ink](https://github.com/inkrs/ink) or [Ratatui](https://github.com/ratatui-org/ratatui)) |
| **LLM Server** | Run model, expose Anthropic-compatible API | **Ollama** (port 11434) or **LM Studio** (port 1234)  [decodesfuture](https://www.decodesfuture.com/articles/how-to-use-a-different-llm-with-claude-code) |
| **Translation Bridge** | Convert Anthropic Messages ↔ Model format + MCP tool routing | **Node.js** with [ollama-mcp-bridge](https://github.com/patruff/ollama-mcp-bridge) pattern  [github](https://github.com/patruff/ollama-mcp-bridge/blob/main/README.md) |
| **Knowledge Repo** | State, history, context storage | JSONL + SQLite in `~/.ai-coder/`  [milvus](https://milvus.io/blog/why-claude-code-feels-so-stable-a-developers-deep-dive-into-its-local-storage-design.md) |

## **4. Step 3: Local Knowledge Repo Structure**

### **4.1 Directory Layout for Maximum Performance**

```bash
~/.ai-coder/
├── settings.json          # Global config (permissions, MCP servers) [web:11]
├── settings.local.json    # Machine-specific (not git-committed) [web:11]
├── history.jsonl          # Command/session history (JSONL for fast append) [web:11]
│
├── projects/              # 📁 Per-project session data
│   ├── <project-name>/
│   │   ├── session.jsonl    # Conversation transcripts
│   │   ├── memory.json      # Auto-generated insights
│   │   └── checkpoints/     # Rollback points
│
├── file-history/          # File edit backups by content hash [web:11]
│   └── <hash>/
│       └── backup.txt
│
├── mcp-config.json        # MCP server definitions [web:16]
├── cache/                 # LLM response cache (for speed)
│   └── <prompt-hash>.json
│
└── index.db               # SQLite full-text index for fast context retrieval
```

### **4.2 Performance Optimizations**

| Technique | Benefit |
|-----------|---------|
| **JSONL format** for history | Fast sequential writes, no parsing overhead  [milvus](https://milvus.io/blog/why-claude-code-feels-so-stable-a-developers-deep-dive-into-its-local-storage-design.md) |
| **SQLite B-tree index** for context | Sub-millisecond keyword search across sessions  [milvus](https://milvus.io/blog/why-claude-code-feels-so-stable-a-developers-deep-dive-into-its-local-storage-design.md) |
| **Content-hash file backups** | O(1) deduplication, instant rollbacks  [milvus](https://milvus.io/blog/why-claude-code-feels-so-stable-a-developers-deep-dive-into-its-local-storage-design.md) |
| **LRU cache** in `cache/` | Skip redundant LLM calls for repeated prompts |

### **4.3 Implementation Code (Python)**

```python
import jsonl
import sqlite3
from pathlib import Path

KNO_PATH = Path.home() / ".ai-coder"

def init_knowledge_repo():
    KNO_PATH.mkdir(exist_ok=True)
    (KNO_PATH / "projects").mkdir(exist_ok=True)
    (KNO_PATH / "file-history").mkdir(exist_ok=True)
    (KNO_PATH / "cache").mkdir(exist_ok=True)
    
    # Initialize SQLite index
    conn = sqlite3.connect(KNO_PATH / "index.db")
    conn.execute("""
        CREATE TABLE IF NOT EXISTS sessions (
            id INTEGER PRIMARY KEY,
            project TEXT,
            prompt TEXT,
            response TEXT,
            timestamp DATETIME
        )
    """)
    conn.commit()

def append_history(project: str, prompt: str, response: str):
    with jsonl.open(KNO_PATH / "history.jsonl", mode="a") as f:
        f.write({"project": project, "prompt": prompt, "response": response, "ts": now()})
```

### **5.3 Filesystem & Shell Tool Implementations**

```python
# Shell executor (Python with asyncio for security)
import asyncio

async def execute_shell_command(command: str, allowed: list) -> str:
    """Secure shell execution using asyncio.create_subprocess_exec"""
    # Parse command to prevent shell injection
    cmd_parts = command.split()
    if cmd_parts[0] not in allowed:
        raise ValueError(f"Command {cmd_parts[0]} not in whitelist")
    
    # Use create_subprocess_exec (safer than shell) [web:22]
    process = await asyncio.create_subprocess_exec(
        cmd_parts[0], *cmd_parts[1:],
        stdout=asyncio.subprocess.PIPE,
        stderr=asyncio.subprocess.PIPE
    )
    stdout, stderr = await process.communicate()
    return stdout.decode() + stderr.decode()

# Filesystem editor
async def edit_file(path: str, content: str) -> bool:
    """Write file with backup"""
    backup_path = Path.home() / ".ai-coder" / "file-history" / f"{hash(content)}.txt"
    backup_path.write_text(path.read_text())  # Backup before edit
    path.write_text(content)
    return True
```

### **5.4 Tool Router (Bridge Core)**

```typescript
// TypeScript (Node.js) bridge pattern from ollama-mcp-bridge [web:16]
class ToolRouter {
  private mcpServers: Map<string, MCPServer>;

  async routeToolCall(toolName: string, input: any): Promise<any> {
    const server = this.mcpServers.get(toolName.split("/")[0]);
    if (!server) throw new Error(`Unknown tool: ${toolName}`);
    
    // JSON-RPC call to MCP server
    return await server.callTool({ name: toolName, arguments: input });
  }
}
```

## **6. Step 5: Technical Stack Recommendations**

### **6.1 Language Choice: Rust vs Python vs Node.js**

| Criteria | **Rust** | **Python** | **Node.js** |
|----------|----------|------------|-------------|
| **Terminal UI performance** | ⭐⭐⭐⭐⭐ (instant startup, no runtime)  [github](https://github.com/Dicklesworthstone/pi_agent_rust) | ⭐⭐ (500ms+ runtime overhead)  [github](https://github.com/Dicklesworthstone/pi_agent_rust) | ⭐⭐⭐ (moderate) |
| **Secure shell execution** | ⭐⭐⭐⭐ (tokio async, no shell) | ⭐⭐⭐⭐ (asyncio.create_subprocess_exec)  [runebook](https://runebook.dev/en/docs/python/library/asyncio-subprocess/asyncio.create_subprocess_shell) | ⭐⭐⭐ (child_process) |
| **Anthropic API libraries** | ⭐⭐ (requastro/anthropic-rust) | ⭐⭐⭐⭐⭐ (`anthropic-python`) | ⭐⭐⭐⭐⭐ (`@anthropic-ai/sdk`) |
| **MCP bridge compatibility** | ⭐⭐ (custom implementation) | ⭐⭐⭐ (ollama-mcp-bridge has Python options) | ⭐⭐⭐⭐⭐ (native TypeScript)  [github](https://github.com/patruff/ollama-mcp-bridge/blob/main/README.md) |
| **Real-time SSE streaming** | ⭐⭐⭐⭐ (hyper + axum) | ⭐⭐⭐ (aiohttp) | ⭐⭐⭐⭐⭐ (express + SSE) |

### **6.2 Recommended Stack: Hybrid Approach**

Based on **oli** (Rust backend + React/Ink frontend)  and **ollama-mcp-bridge** (TypeScript): [github](https://github.com/amrit110/oli)

| Layer | Technology | Libraries |
|-------|------------|-----------|
| **CLI Client** | **Rust** | [Ratatui](https://github.com/ratatui-org/ratatui) (TUI), [reqwest](https://github.com/reqwest/reqwest) (HTTP), [tokio](https://github.com/tokio-rs/tokio) (async)  [github](https://github.com/Dicklesworthstone/pi_agent_rust) |
| **Bridge/Proxy** | **Node.js (TypeScript)** | [@anthropic-ai/sdk](https://github.com/anthropic-ai/anthropic-js), [express](https://github.com/expressjs/express), [ollama-mcp-bridge](https://github.com/patruff/ollama-mcp-bridge) pattern  [github](https://github.com/patruff/ollama-mcp-bridge/blob/main/README.md) |
| **Shell Executor** | **Rust** (embedded in CLI) | `tokio::process` (async subprocess), no shell injection  [github](https://github.com/Dicklesworthstone/pi_agent_rust) |
| **Knowledge Repo** | **Python** (utility scripts) | `jsonl`, `sqlite3`, `pathlib`  [milvus](https://milvus.io/blog/why-claude-code-feels-so-stable-a-developers-deep-dive-into-its-local-storage-design.md) |
