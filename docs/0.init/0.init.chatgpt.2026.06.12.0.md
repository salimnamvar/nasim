# 🔌 2. API Compatibility Layer (Anthropic Native Emulation)

## 🎯 Goal

Make your system behave like:

```
https://api.anthropic.com/v1/messages
```

But internally route to:

* Ollama `/api/chat`
* LM Studio `/v1/chat/completions`
* or any OpenAI-compatible endpoint

---

## 📦 Key Design Choice

Implement a **“Claude API Shim”**:

### Endpoint Mapping

| Anthropic           | Your Server      |
| ------------------- | ---------------- |
| POST `/v1/messages` | `/v1/messages`   |
| streaming SSE       | streaming SSE    |
| tools               | tool router      |
| system prompt       | system injection |

---

## 🧩 Request Translation Layer

Convert:

```json
Anthropic Messages Format
```

into:

```json
Ollama / OpenAI Chat Format
```

### Example mapping:

```ts
function anthropicToOpenAI(req) {
  return {
    model: mapModel(req.model),
    messages: [
      { role: "system", content: req.system },
      ...req.messages
    ],
    stream: req.stream
  }
}
```

---

## 🔁 Response Normalization

Wrap backend output back into Anthropic format:

```ts
{
  id: crypto.randomUUID(),
  type: "message",
  role: "assistant",
  content: [
    { type: "text", text: response }
  ]
}
```

---

## ⚡ Streaming Layer (Critical)

Use:

* SSE (Server-Sent Events)
* or WebSockets for lower latency

Node example:

* `eventsource-parser`
* `fastify-sse-v2`

# 🧰 5. Tool Use & Bridge System (Claude-style agent tools)

This is the **most important part** of your system.

---

## 🎯 Goal

Allow LLM to execute:

* filesystem edits
* shell commands
* git operations
* search/indexing

---

## 🧱 Architecture: Tool Router (“Bridge”)

```
LLM → tool_call → tool_router → executor → result → LLM
```

---

## 📦 Tool Schema (Anthropic-compatible)

```json
{
  "name": "bash",
  "description": "Run shell command",
  "input_schema": {
    "type": "object",
    "properties": {
      "command": { "type": "string" }
    }
  }
}
```

---

## ⚙️ Tool Executor Layer

### Safe shell execution

Use:

* Node: `execa`
* Python: `subprocess + pty`
* Rust: `tokio-process`

---

### 🔐 Security sandbox

Add:

* allowlist commands
* deny destructive ops
* path restrictions

Example:

```ts
const ALLOWED = ["ls", "cat", "git", "npm", "node"]

function validate(cmd) {
  return ALLOWED.some(x => cmd.startsWith(x))
}
```

---

## 🧠 Tool loop logic

```ts
while (toolCall) {
  const result = await runTool(toolCall)
  messages.push({
    role: "tool",
    content: result
  })

  continueLLM()
}
```

---

## 🧩 Bridge Enhancements (Claude-like intelligence)

Add special tools:

### 1. file_edit

* diff-based editing
* patch application

### 2. search_repo

* ripgrep wrapper
* symbol index lookup

### 3. plan_mode

* non-executing reasoning step

# 🔐 7. Secure Shell Execution Design

## ⚠️ Critical requirement

Never allow raw unrestricted shell execution.

---

## 🧱 Sandboxing Layers

### Layer 1: Command filter

* regex allowlist
* deny rm -rf /

### Layer 2: Working directory sandbox

```ts
process.chdir("/workspace/project")
```

### Layer 3: timeout enforcement

```ts
timeout: 5000ms
```

### Layer 4: interactive isolation

Use PTY:

* `node-pty`
* prevents escape control
