## 1. What Nasim Is

Nasim is a renamed, locally-routed fork of Claude Code
(`https://github.com/anthropics/claude-code`). It is not a reimplementation. It is
the exact Claude Code TypeScript source with:

1. All `claude` entity names replaced with `nasim` at every level — files,
   directories, code identifiers, environment variables, config keys, documentation.
2. The LLM API endpoint redirected from `api.anthropic.com` to a Bridge service
   running on `black` (LAN machine), which proxies to Ollama.
3. A sync script that pulls the latest upstream Claude Code release, re-applies the
   rename transformation, and re-applies Nasim-specific patches — keeping Nasim
   permanently current with Claude Code's capabilities.

Nasim's knowledge base (`~/.nasim/`, `NASIM.md`) is completely separate from Claude
Code's (`~/.claude/`, `CLAUDE.md`). Running both on the same machine is safe and
intentional — they share no state, no config, no history.

## 2. Architecture

```
salim-hp (client)                           black (server)
────────────────────────────────            ────────────────────────────────
nasim CLI                                   Bridge (:8080)
  └── SSH tunnel                              ├── server.py   (FastAPI proxy)
      localhost:8080 → black:8080             └── translator.py  (format adapter)
                  │                                       │
                  │  Anthropic Messages API               │
                  └──────────────────────────────────────►│
                                                          ▼
                                                    Ollama (:11434)
                                                      └── qwen2.5-coder:32b
                                                          qwen3-coder (backup)
```

**Client (salim-hp):**
- Nasim CLI (renamed Claude Code TypeScript build)
- Runs the agentic tool loop locally — tools operate on salim-hp filesystem
- SSH tunnel forwards `localhost:8080` → `black:8080`
- `ANTHROPIC_BASE_URL=http://localhost:8080` redirects all SDK calls through tunnel

**Server (black):**
- Ollama serving the LLM (no internet required after model pull)
- Bridge: a minimal FastAPI service (~150 lines) that accepts Anthropic Messages API
  format and proxies to Ollama's `/v1/messages` endpoint
- Exposes port 8080 locally on black (not internet-accessible; SSH-only)

**Why loop-in-client (validated in 2.val):**
Tools (`read_file`, `write_file`, `bash`) must operate on salim-hp's filesystem.
SSH tunnel adds ~2ms per LLM call, which is negligible against 1–30s inference time.
The Bridge is a pure proxy — no loop logic, no tool execution, no state.

## 4. Repository Strategy

### 4.1 Structure

```
nasim/                         ← this repo (salimnamvar/nasim on GitHub)
├── upstream/                  ← git subtree: anthropics/claude-code (read-only)
│   └── ...claude-code source
├── src/                       ← transformed source (generated by sync script)
│   └── ...nasim source
├── bridge/                    ← Bridge service for black (Nasim-specific, not in Claude Code)
│   ├── server.py
│   └── translator.py
├── scripts/
│   ├── sync-from-upstream.sh  ← THE KEY SCRIPT (see Section 6)
│   ├── install-server.sh      ← installs Bridge + Ollama on black via SSH
│   └── install-client.sh      ← installs Nasim CLI on salim-hp
├── patches/                   ← Nasim-specific patches applied after rename
│   ├── 001-api-redirect.patch ← wires NASIM_API_URL as default base URL
│   ├── 002-model-map.patch    ← adds Claude→Ollama model name mapping
│   └── 003-rename-config-paths.patch ← ~/.nasim/ paths
├── NASIM.md                   ← global Nasim context (equivalent of ~/.claude/CLAUDE.md)
└── pyproject.toml             ← Bridge dependencies (Python)
```

### 4.2 Why Not a Pure Git Fork

A standard git fork of Claude Code would accumulate merge conflicts every time
Claude Code releases a new version, because every file has been renamed. A sync
script that re-applies the transformation from scratch is cleaner:

- No merge conflicts — each sync starts from clean upstream
- Patches are explicit, version-controlled, and auditable
- Nasim-specific features live in `patches/` as numbered diffs
- A new Claude Code release becomes: `./scripts/sync-from-upstream.sh && npm test`

### 5.4 Test Ollama Anthropic Endpoint

This test determines whether the translation layer in the Bridge is needed:

```bash
ssh black "curl -s http://localhost:11434/v1/messages \
  -H 'Content-Type: application/json' \

    \"model\": \"qwen2.5-coder:32b\",
```

Expected: JSON response with `content[0].type == "text"`. If the endpoint returns

Test tool calling (determines if translator.py is needed at all):

```bash

  -H 'Content-Type: application/json' \
  -d '{
    \"model\": \"qwen2.5-coder:32b\",
    \"max_tokens\": 256,
    \"tools\": [{
      \"name\": \"read_file\",
      \"description\": \"Read a file\",
      \"input_schema\": {
        \"type\": \"object\",
        \"properties\": {\"path\": {\"type\": \"string\"}},
        \"required\": [\"path\"]
      }
    }],
    \"messages\": [{\"role\": \"user\", \"content\": \"read src/main.py\"}]
  }'"
```

Expected: response with `content[0].type == "tool_use"` and `content[0].input.path == "src/main.py"`.

Record the result — it determines whether Phase 1 Bridge needs translator.py active or in
pass-through mode.

## 6. The Sync Script (`scripts/sync-from-upstream.sh`)

This is the core maintenance script. It transforms the latest Claude Code source into
Nasim. Run it whenever a new Claude Code version is released.

### 6.1 What It Does

```
1. Fetch latest claude-code tag from GitHub
2. Export the source tree (no git history)
3. Apply rename transformations (directories, files, content)
4. Apply Nasim-specific patches from patches/
5. Build (npm install + tsc)
6. Run test suite
7. Report diff from previous Nasim version
```

### 6.2 Script Design

```bash
#!/usr/bin/env bash
# scripts/sync-from-upstream.sh
# Usage: ./scripts/sync-from-upstream.sh [--version v1.2.3] [--dry-run]
set -euo pipefail

UPSTREAM_REPO="https://github.com/anthropics/claude-code.git"
UPSTREAM_VERSION="${1:-}"          # empty = latest tag
SRC_DIR="src"
PATCHES_DIR="patches"
WORK_DIR="$(mktemp -d)"

# ── 1. Fetch upstream ────────────────────────────────────────────────
echo "→ Fetching upstream claude-code..."
git clone --depth 1 ${UPSTREAM_VERSION:+--branch "$UPSTREAM_VERSION"} \
    "$UPSTREAM_REPO" "$WORK_DIR/upstream"
UPSTREAM_TAG=$(git -C "$WORK_DIR/upstream" describe --tags --abbrev=0 2>/dev/null || echo "unknown")
echo "  upstream version: $UPSTREAM_TAG"

# ── 2. Clean previous src ────────────────────────────────────────────
echo "→ Cleaning $SRC_DIR..."
rm -rf "$SRC_DIR"
cp -r "$WORK_DIR/upstream" "$SRC_DIR"
rm -rf "$SRC_DIR/.git"

# ── 3. Rename directories ────────────────────────────────────────────
echo "→ Renaming directories..."
# Process deepest first to avoid renaming parent before children
find "$SRC_DIR" -depth -type d -name "*claude*" | while read -r dir; do
    new=$(echo "$dir" | sed \
        -e 's/claude-code/nasim/g' \
        -e 's/claude_code/nasim/g' \
        -e 's/claudeCode/nasim/g' \
        -e 's/claude/nasim/g')
    [[ "$dir" != "$new" ]] && mv "$dir" "$new"
done

# ── 4. Rename files ──────────────────────────────────────────────────
echo "→ Renaming files..."
find "$SRC_DIR" -type f -name "*claude*" | while read -r file; do
    new=$(echo "$file" | sed \
        -e 's/CLAUDE\.md/NASIM.md/g' \
        -e 's/claude-code/nasim/g' \
        -e 's/claude_code/nasim/g' \
        -e 's/claudeCode/nasim/g' \
        -e 's/claude/nasim/g')
    [[ "$file" != "$new" ]] && mv "$file" "$new"
done

# ── 5. Rename content ────────────────────────────────────────────────
echo "→ Renaming content in files..."
find "$SRC_DIR" -type f \( \
    -name "*.ts" -o -name "*.js" -o -name "*.json" -o \
    -name "*.md" -o -name "*.yaml" -o -name "*.yml" -o \
    -name "*.sh" -o -name "*.env*" -o -name "*.txt" -o \
    -name "Makefile" -o -name "Dockerfile" \) \
| xargs -I{} sed -i \
    -e 's/CLAUDE\.md/NASIM.md/g' \
    -e 's/~\/\.claude\//~\/.nasim\//g' \
    -e 's/\.claude\//.nasim\//g' \
    -e 's/@anthropic-ai\/claude-code/nasim/g' \
    -e 's/ANTHROPIC_API_KEY/NASIM_API_KEY/g' \
    -e 's/ANTHROPIC_BASE_URL/NASIM_API_URL/g' \
    -e 's/CLAUDE_CODE_/NASIM_/g' \
    -e 's/ClaudeCode/NasimCode/g' \
    -e 's/claudeCode/nasimCode/g' \
    -e 's/claude-code/nasim/g' \
    -e 's/claude_code/nasim/g' \
    -e 's/Claude Code/Nasim/g' \
    -e 's/CLAUDE CODE/NASIM/g' \
    -e 's/"claude"/"nasim"/g' \
    -e 's/'\''claude'\''/'\''nasim'\''/g' \
    {}

# ── 6. Apply Nasim-specific patches ─────────────────────────────────
echo "→ Applying patches..."
for patch in "$PATCHES_DIR"/*.patch; do
    [[ -f "$patch" ]] || continue
    echo "  applying: $(basename "$patch")"
    patch -d "$SRC_DIR" -p1 < "$patch"
done

# ── 7. Record upstream version ───────────────────────────────────────
echo "UPSTREAM_CLAUDE_CODE_VERSION=$UPSTREAM_TAG" > "$SRC_DIR/.upstream-version"
echo "NASIM_SYNC_DATE=$(date -u +%Y-%m-%dT%H:%M:%SZ)" >> "$SRC_DIR/.upstream-version"

# ── 8. Build ─────────────────────────────────────────────────────────
echo "→ Building..."
cd "$SRC_DIR" && npm install && npm run build

echo "✓ Sync complete. Upstream: $UPSTREAM_TAG"
rm -rf "$WORK_DIR"
```

### 6.3 Patches Directory

Each patch in `patches/` is a standard unified diff applied after the rename.
Patches are numbered and ordered by application sequence.

| File                              | Purpose                                            |
|-----------------------------------|----------------------------------------------------|
| `001-api-redirect.patch`          | Default `NASIM_API_URL=http://localhost:8080`       |
| `002-model-map.patch`             | Claude model names → Ollama model names in config  |
| `003-config-paths.patch`          | Hardcode `~/.nasim/` as config root                |
| `004-systemd-tunnel.patch`        | Add SSH tunnel setup to installation               |
| `005-version-banner.patch`        | Replace "Claude Code" brand in version output      |

Generate a patch from a manual edit:
```bash
git diff src/ > patches/006-my-feature.patch
```

### 6.4 Running the Sync

```bash
# Sync to latest Claude Code release
./scripts/sync-from-upstream.sh

# Sync to a specific version
./scripts/sync-from-upstream.sh --version v1.2.3

# After sync: test and commit
cd src && npm test
cd ..
git add src/ patches/
git commit -m "chore(sync): update from claude-code $UPSTREAM_TAG"
```

## 7. Bridge Service

The Bridge is ~150 lines of Python. It lives in `bridge/` and runs on `black`.
It is NOT part of the Claude Code fork — it is Nasim-original code.

### 7.1 `bridge/server.py`

```python
"""Bridge: Anthropic Messages API proxy to Ollama."""

import os
import httpx
from fastapi import FastAPI, Request, HTTPException
from fastapi.responses import StreamingResponse

app = FastAPI(title="Nasim Bridge")

OLLAMA_URL = os.environ.get("OLLAMA_URL", "http://localhost:11434")
DEFAULT_MODEL = os.environ.get("DEFAULT_MODEL", "qwen2.5-coder:32b")

MODEL_MAP = {
    "claude-opus-4-8":           "qwen2.5-coder:32b",
    "claude-sonnet-4-6":         "qwen2.5-coder:32b",
    "claude-haiku-4-5-20251001": "qwen2.5-coder:7b",
    "claude-fable-5":            "qwen3-coder",
}


@app.get("/health")
async def health():
    try:
        async with httpx.AsyncClient(timeout=3) as c:
            r = await c.get(f"{OLLAMA_URL}/api/tags")
        return {"status": "ok", "ollama": "connected", "model": DEFAULT_MODEL}
    except Exception as e:
        return {"status": "degraded", "error": str(e)}


@app.post("/v1/messages")
async def messages(request: Request):
    body = await request.json()

    # Map model name
    requested = body.get("model", DEFAULT_MODEL)
    body["model"] = MODEL_MAP.get(requested, DEFAULT_MODEL)

    # Forward to Ollama native /v1/messages
    url = f"{OLLAMA_URL}/v1/messages"

    if body.get("stream", False):
        return StreamingResponse(
            _stream(url, body),
            media_type="text/event-stream",
        )

    async with httpx.AsyncClient(timeout=120) as c:
        r = await c.post(url, json=body)
        if r.status_code != 200:
            raise HTTPException(status_code=r.status_code, detail=r.text)
        return r.json()


async def _stream(url: str, body: dict):
    async with httpx.AsyncClient(timeout=120) as c:
        async with c.stream("POST", url, json=body) as r:
            async for chunk in r.aiter_bytes():
                yield chunk
```

## 9. Phase Roadmap

Each phase ends with a testable milestone. Phases are sequential — do not start a
phase until the previous phase's test passes.

---

### Phase 0 — Server Preparation (black)

**Goal:** Ollama running and Bridge responding on black.

Steps (from Section 5):
1. SSH access verified
2. Ollama installed and running
3. `qwen2.5-coder:32b` pulled
4. Ollama native `/v1/messages` tested — record whether tool calling works
5. Python environment installed on black
6. Bridge service installed and running as systemd service

**Exit test:**
```bash
# From salim-hp:
ssh black "curl -s http://localhost:8080/health"
# Expected: {"status": "ok", ...}
```

**Estimated time:** 2–4 hours (GPU model pull is the bottleneck)

---

### Phase 1 — Nasim CLI Running Locally

**Goal:** `nasim "hello"` streams a response from black.

Steps:
1. Fork Claude Code: `./scripts/sync-from-upstream.sh`
2. Verify rename transformations are correct (audit 10 files manually)
3. Build: `cd src && npm install && npm run build`
4. Create SSH tunnel alias
5. Set `~/.nasim/settings.json`
6. Set `~/.nasim/NASIM.md`
7. Install: `npm install -g .`

**Exit test:**
```bash
nasim "reply with exactly: nasim phase 1 complete"
# Expected: streaming text output: "nasim phase 1 complete"
```

Manual checks:
- `nasim --version` shows "Nasim" not "Claude Code"
- `~/.nasim/` directory created (not `~/.claude/`)
- Project context reads from `NASIM.md` (not `CLAUDE.md`)

**Estimated time:** 1–2 hours

---

### Phase 2 — Tool Loop Verified

**Goal:** Nasim executes agentic tool calls — reads files, writes edits, runs bash.

Steps:
1. Create a test project with known files
2. Run a tool-using query
3. Verify blocklist enforcement (blocked patterns must be rejected)
4. Verify confirmation gate works for destructive commands

**Exit test:**
```bash
# In a test directory with a Python file:
nasim "add a docstring to every function in test.py"
# Expected: Nasim reads test.py, edits it, shows diff
```

Additional tests:
```bash
# Blocklist test — must be blocked without confirmation:
nasim "run: rm -rf /"
# Expected: blocked with security error, command NOT executed
```

### Phase 6 — Production Hardening

**Goal:** Nasim is reliable enough to be the daily driver.

Steps:
1. Bridge systemd service auto-restarts on failure
2. SSH tunnel reconnects if dropped (use `autossh`)
3. `nasim-bridge` service survives `black` reboot — model is reloaded automatically
4. Model falls back to `qwen2.5-coder:7b` if 32b is OOM
5. Token usage display in session summary
6. Error messages are clear and actionable (not raw API errors)

**Autossh tunnel (replace simple ssh alias):**
```bash
# ~/.bashrc
export NASIM_API_URL=http://localhost:8080
export NASIM_API_KEY=local

# Start persistent tunnel (survives disconnects)
nasim-tunnel() {
    autossh -M 0 -f -N \
        -o "ServerAliveInterval=30" \
        -o "ServerAliveCountMax=3" \
        -L 8080:localhost:8080 black
}

# Auto-start tunnel if not running
alias nasim='pgrep -f "L 8080:localhost:8080" > /dev/null || nasim-tunnel; command nasim'
```

**Exit test:**
```bash
# Kill the tunnel and verify nasim reconnects:
pkill -f "L 8080:localhost:8080"
nasim "hello"    # should auto-reconnect and respond
```

**Estimated time:** 3–4 hours
