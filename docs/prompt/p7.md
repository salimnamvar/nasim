**🚨 CRITICAL — SMT ownership violation in Session SM**

`sm_session_lifecycle.puml` has two transitions writing to the `ACTIVE` state:

```
CREATED  → ACTIVE   : SSN-01 session initialized
RESTORED → ACTIVE   : SSN-04 session resumed
```

But the SM ownership table in `docs/SM/README.md` assigns `ACTIVE` exclusively to `SSN-01 PERSIST Session`. **Direct contradiction of the documented "one lifecycle-write UC per target state" rule.**

**Recommended fix (smaller blast radius):** re-label `RESTORED → ACTIVE` to route through SSN-01 — same pattern already used for `RESTORED → SAVED : SSN-01 save after restore`. Don't weaken the rule to "at most two writers".

**✅ Process vs entity distinction correct.** `sm_agent_lifecycle.puml` carries the explicit deviation comment, and the README reaffirms: Agent is a process FSM (17 transient states), Session/Plan/Plugin are entity lifecycles. Exemption is appropriate.

**✅ Hex palette consistent across all 4 SMs.** `#ECEFF1` for IDLE/EMPTY/DISCOVERED, `#2E7D32` for ACTIVE/ENABLED/APPROVED, etc. — same color for the same conceptual state.

**🚨 CRITICAL — Phantom UC reference**

`uc_server.puml` line 317:
```
usecase "SSN-09\nDELETE Session\n[uc_session]" as SSN09_ext <<extref>>
```
Line 367:
```
SRV05 ..> SSN09_ext : <<include>>
```

But the SSN group has only `SSN-01` through `SSN-08`. **`SSN-09` does not exist.** This is a hard failure — the diagram references a non-existent identifier.

**Recommended fix:** add `SSN-09 DELETE Session` to the SSN group (owner: `SessionStore`) — the absence of a delete operation is a real gap in the SSN API surface, not just a doc bug.

**🟡 M3 — Fork duplication.** `SSN-08 BRANCH Session` (owner: `SessionFork`) and `WRL-04 FORK Session` (owner: `SessionForkManager`) describe essentially the same operation with two different owners and two different C4 components. The SM resolves the conflict by assigning `BRANCHED` ownership to `WRL-04` only, but the inventory/UC layer doesn't disambiguate. Pick one canonical path.

**🔵 L1 — Agent SM transitions without UC labels** for non-user actions:
- `THINKING → COMPACTING : token_count > budget` — should reference `AGT-06`
- `THINKING → SERVING : Response to HTTP client` — should reference `SRV-06`
