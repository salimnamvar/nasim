# Comprehensive Audit of nasim C4 → UC → SM Consistency

## Overall Assessment

The updated design introduces **significant architectural improvements**:

- **New groups**: `Safety` and `Router` now have explicit boundaries, improving separation of concerns.
- **Layering**: The core library no longer depends on its callers (`Rel(agent_orch, cli_container)` and `Rel(agent_orch, server_container)` removed) – fixes dependency inversion.
- **UC cross‑referencing**: Use of `<<extref>>` in `uc_server.puml` and `uc_cli.puml` eliminates duplication and clearly delegates to internal groups.

However, the transition is **incomplete**. Several inconsistencies between the C4 component inventory, the actual diagram files, and the UC/SM models remain. The table below summarises the **critical issues** that break traceability and architectural coherence.

## CAR (Challenge‑Action‑Result) Prompt for Autonomous Fix

The following tasks must be executed in order to fully harmonise the C4, UC, and SM models.

---

### CAR‑C4‑01: Synchronise C4 Component Inventory with Diagrams

**Challenge (Measure)**
The C4 inventory (`docs/C4/README.md`) is outdated and does not reflect the actual components present in the diagram files or the newly introduced groups (`Safety`, `Router`). Specifically:
- `PlanSession`, `CompactionPolicy`, `QualitySignal`, `ConfigSchema` are missing.
- `ModelRouter`, `FallbackChain`, `ProviderCapabilities` are incorrectly still in the Provider group inventory and diagram.
- `SafetyCoordinator` is incorrectly still in the Agent group diagram.

**Action (Fix)**
1. **Update `docs/C4/README.md` (Component Inventory)**:
   - **Agent group**: Add `PlanSession` and `CompactionPolicy`. Remove `SafetyCoordinator` (it belongs to Safety).
   - **Provider group**: Remove `ModelRouter`, `FallbackChain`, `ProviderCapabilities`. Ensure only `Provider (Protocol)` and `LiteLLMProxy` remain.
   - **Config group**: Add `ConfigSchema` (the typed configuration dataclass).
   - **Evaluation group**: Add `QualitySignal`.
   - **Subagent group**: **Delete** the entire Subagent row (to avoid duplication – `SubagentCoordinator` is already in Agent).
2. **Edit the affected C4 component diagrams** to match the updated inventory:
   - **`c4_nasim_component_agent.puml`**: Remove `SafetyCoordinator`. Ensure `PlanSession` and `CompactionPolicy` are present.
   - **`c4_nasim_component_provider.puml`**: Remove `ModelRouter`, `FallbackChain`, `ProviderCapabilities` (they are now in Router).
   - **`c4_nasim_component_config.puml`**: Keep `ConfigSchema` (already present).
   - **`c4_nasim_component_evaluation.puml`**: Add `QualitySignal` component.
   - **`c4_nasim_component.puml` (overview)**: Update the overview to include the new `Safety` and `Router` groups, and reflect the component changes in each group.
   - **Delete `c4_nasim_component_subagent.puml`** (or remove it from the README and the overview, as it duplicates Agent).

**Result (Verify)**
- `grep -E "PlanSession|CompactionPolicy" docs/C4/README.md` returns both.
- `grep "QualitySignal" docs/C4/README.md` returns it.
- `grep "ConfigSchema" docs/C4/README.md` returns it.
- The Provider group in README no longer lists Router components.
- The Subagent group is gone.
- The overview diagram contains only the 18 unique groups (including Safety and Router) with their correct components.

### CAR‑C4‑03: Resolve `EXECUTING → COMPLETED` Transition in Plan SM

**Challenge (Measure)**
The Plan lifecycle SM uses `AGT-08` (APPROVE Plan) as the trigger for the `EXECUTING --> COMPLETED` transition. However, `AGT-08` is the approval step; completion happens implicitly when the agent finishes executing all queued tool calls, which is driven by `AGT-01 PROCESS User Task`.

**Action (Fix)**
1. **Edit `docs/SM/sm_plan_lifecycle.puml`**.
2. Change the transition:
   ```
   EXECUTING --> COMPLETED : AGT-08 all steps done
   ```
   to:
   ```
   EXECUTING --> COMPLETED : AGT-01 all steps executed
   ```
3. Update the **Plan Lifecycle** table in `docs/SM/README.md` to reflect that `COMPLETED` is written by `AGT-01` (the agent loop), not `AGT-08`.

**Result (Verify)**
- `grep "COMPLETED" docs/SM/sm_plan_lifecycle.puml` shows `AGT-01`.
- The SMT mapping table for Plan is updated accordingly.
