## 2. Remaining Gap: Missing `SSN-09 DELETE Session` in UC Inventory

**Observation (Measure)**  
In `uc_server.puml`, the `SSN-09_ext` external reference is declared and used by `SRV-05 DELETE Session`:
```
usecase "SSN-09\nDELETE Session\n[uc_session]" as SSN09_ext <<extref>>
...
SRV05 ..> SSN09_ext : <<include>>
```

However, `SSN-09` is **not listed** in the UC inventory (`docs/UC/README.md`). The inventory currently ends at `SSN-08 BRANCH Session`. This creates a **traceability gap** between the server boundary use case and the internal session deletion operation.

**Impact**  
- The inventory is incomplete, breaking the "one authoritative source" rule.
- A reader cannot find `SSN-09` in the expected place, reducing the model's usability.

**Root Cause**  
During the refactoring of `uc_server.puml`, `SRV-05` was renamed from `RETIRE Session` to `DELETE Session`, and the corresponding internal UC `SSN-09` was introduced in the diagram but **not added to the inventory table**.

**Recommendation**  
Add `SSN-09 DELETE Session` to the UC inventory (under the `SSN` group), with the component owner `SessionStore`.
