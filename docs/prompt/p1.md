# Final Audited Design Report: C4 → UC → SM Consistency & Architectural Integrity

## 1. Executive Assessment

Your **C4** model (Structural) has reached **10/10** status. The **UC** (Behavioural) and **SM** (State Machine) models are also highly comprehensive.

However, the audit reveals **one severe architectural anti-pattern** (layering violation) and several **minor consistency gaps** between the State Machines and the Use Cases.

**Central Finding**: The `Developer` actor is incorrectly modeled as directly invoking internal core library Use Cases (`MEM-02 RECALL Knowledge`, `SSN-07 SEARCH Sessions`, etc.). In reality, the Developer only interacts with the **CLI** or **HTTP Server** boundaries. The **Agent** is the sole orchestrator of internal domain logic. This violates the **Controller-Service-Repository** (or Hexagonal/Clean Architecture) separation of concerns.

---

## 2. Deep-Dive Audit Findings

### A. The "Suspicious" MEM02 Actor Issue (Layering Violation)
- **Issue**: In `uc_memory.puml` and `uc_overview.puml`, the `Developer` actor has a direct line to `MEM-02 RECALL Knowledge`.
- **Why it’s wrong**: In the C4 model, `Memory` is a Core Library component. The `Developer` sends input to the CLI, which passes it to `AgentOrchestrator`. The Agent decides when to recall memory via `TL-20 RECALL Memory` (Tool), which internally invokes `MEM-02`. Exposing `MEM-02` directly to the Developer bypasses the Agent's governance (permissions, context injection, persona switching).
- **Fix**: Remove all direct `Developer` links to internal groups (`MEM`, `SSN`, `CTX`, `RIM`, `EDT`, `EVL`, `PRV`, `RTG`). These must be orchestrated solely by the `Agent` actor.

### B. Overview UC Diagram ("Massive Mesh")
- **Question**: Should we have a massive all-in-one UC diagram?
- **Verdict**: **No**. A single diagram with all 130+ UCs would be illegible (spaghetti). The current package-based overview (showing representative UCs per group) is standard practice. I will recommend a **cross-reference matrix** or a **layered sequence map** if they want to visualize complex cross-group interactions, but a massive mesh is anti-pattern.

### C. SM → UC Coverage (SMT Ownership)
- **SMT Rule Check**: The `SM/README.md` correctly states "one lifecycle-write UC per target state".
- **Session**: `CREATED` (SSN-01), `SAVED` (SSN-01), `RESTORED` (SSN-04), `BRANCHED` (WRL-04) — all have explicit UCs.
- **Plan**: `BUILDING`/`QUEUED` (AGT-07), `APPROVED` (AGT-08), `EXECUTING` (AGT-08). `COMPLETED` is implicit when the Agent loop finishes (AGT-01). This is acceptable, though we can add a note linking it to AGT-01.
- **Agent**: `STAGING` is triggered by `EDT-10`. `AWAITING_DIFF_APPROVAL` by `SAF-02`. All transitions are covered.
- **Missing SM for Subagent?**: No, Subagents are process-instances of the same Agent FSM. The design is recursive and correct.
- **Conclusion**: SM coverage is solid.
