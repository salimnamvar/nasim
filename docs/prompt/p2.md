## 2. Detailed Findings

### 2.1 Session Lifecycle: Improper Use of `AGT-14` for Normal Closure

**Observation**  
In `sm_session_lifecycle.puml`:
- `ACTIVE --> CLOSED : AGT-14 /quit or explicit close`
- `SAVED --> CLOSED : AGT-14 explicit close after save`

`AGT-14` is defined in the UC inventory as `HANDLE Error` – it is an error recovery use case. Using it to terminate a session normally conflates error handling with normal shutdown. This violates the **single responsibility** principle and obscures the true UC that closes sessions.

**Root cause**  
There is no dedicated UC for gracefully closing a session. The actual closing is triggered by the user (via `/quit` in CLI or `DELETE /sessions/{id}` in HTTP) or by the system when the process exits.

**Impact**  
- Confuses readers: an error handler appears to be responsible for normal termination.
- Makes it harder to trace the real behaviour in sequence diagrams.

**Recommendation**  
Introduce a new UC (e.g., `CLI-09 CLOSE Session` and `SRV-12 CLOSE Session`) or reuse an existing boundary UC. Map the `CLOSED` state to that UC.

---

### 2.2 Missing Boundary‑to‑Internal Traceability for CLI Slash Commands

**Observation**  
- `CLI-07 SWITCH Model` (Developer → CLI‑07) currently has no `<<include>>` to the internal use case that performs the switch (`RTG-04 SWITCH Model`).
- `CLI-05 ENABLE Plan Mode` (Developer → CLI‑05) has no `<<include>>` to `AGT-07 QUEUE Plan` (or `AGT-08 APPROVE Plan`).

**Impact**  
Traceability from the user action to the internal domain operation is broken. A reader cannot see that `/switch_model` invokes the router, or that `/plan` queues a plan.

**Recommendation**  
Add explicit `<<include>>` relationships in `uc_cli.puml`:
```
CLI07 ..> RTG04 : <<include>>
CLI05 ..> AGT07 : <<include>>
```

### 2.3 Missing UC ID Labels on Some Agent SM Transitions

**Observation**  
In `sm_agent_lifecycle.puml`, the transition:
```
IDLE --> SERVING : HTTP request received (server mode)
```
lacks a UC ID. The convention (stated in the diagram header) is that every transition should be annotated with its triggering UC. Similarly, some internal transitions (e.g., `THINKING --> EVALUATING`) have UC IDs, but this one does not.

**Impact**  
Inconsistent with the stated convention; reduces the diagram’s value as a quick reference.

**Recommendation**  
Add the UC ID explicitly, e.g.:
```
IDLE --> SERVING : SRV-06 HTTP request received
```

---

### 2.4 Duplication of UC Definitions in `uc_server.puml`

**Observation**  
In `uc_server.puml`, the `rectangle "nasim"` block re‑declares UCs from other groups (`SSN-01`, `AGT-01`, `CFG-01`) inside the Server package solely for the purpose of showing `<<include>>` relationships. This creates duplicate `usecase` elements with the same ID, which may cause rendering issues in PlantUML (duplicate definition) and confuses the model.

**Impact**  
- Potential PlantUML syntax error (duplicate IDs).
- Blurs the grouping: Server UCs should only be the boundary SRV‑XX UCs. Internal UCs should be referenced via `Container_Ext` or by referencing the existing package.

**Recommendation**  
Remove the duplicated UCs and instead use a `note` or a separate `package` definition that is not re‑declared, or reference the existing UCs using `usecase` aliases without re‑defining. Better yet, use `' include` comments in the `Rel` lines to indicate the delegation, as PlantUML allows linking to existing UCs by their alias if they are defined elsewhere. But the simplest fix is to remove the internal UC definitions from `uc_server.puml` and keep only the SRV UCs, and then use `Rel` with `->` or `..>` to show the delegation, e.g., `SRV01 ..> SSN03 : <<include>>` where `SSN03` is defined in `uc_session.puml`. However, if they are in separate files, we need to `!include` the session file or use a global variable. Given the current design, they likely include all files in the overview; for individual diagrams, it’s better to avoid duplication. We can simply use `note` or `Rel` to an external component symbol (Container_Ext) to show delegation.

**Recommendation**  
Refactor `uc_server.puml` to not duplicate UCs; instead, use `Container_Ext` for the core library groups and show dependency via `Rel` from SRV UCs to those `Container_Ext` elements. This maintains traceability without duplication.
