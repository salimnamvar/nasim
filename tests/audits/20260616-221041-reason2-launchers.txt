To address the issue of "models are not working with the clis" even after a correct tunnel + good tag, we need to ensure that each launcher correctly sets environment variables, handles model names appropriately, and maintains the model argument throughout execution chains.

### claude.sh

**Issues:**
- The script might be missing an export for the `ANTHROPIC_BASE_URL` or `AUTH_TOKEN`.

**Recommended Patch:**

```sh
#!/bin/bash

# Set the base URL and auth token for Anthropic compatibility
export ANTHROPIC_BASE_URL="http://localhost:1234"
export AUTH_TOKEN="ollama"

# Execute the claude CLI with the model argument
exec claude "$@"
```

### aider.sh

**Issues:**
- The script might be missing an export for `OLLAMA_API_BASE`.
- The model name might need to be prefixed with `ollama/`.

**Recommended Patch:**

```sh
#!/bin/bash

# Set the Ollama API base URL
export OLLAMA_API_BASE="http://localhost:1234"

# Execute the aider CLI with the ollama/<tag> prefix for the model argument
exec aider "ollama/$@"
```

### opencode.sh

**Issues:**
- The script might be missing an export for `url`.
- The model name might need to be prefixed with `ollama/` if using the `ollama launch` command.

**Recommended Patch:**

```sh
#!/bin/bash

# Set the OpenAI compatible URL or use ollama launch
export url="http://localhost:1234"

# Execute the opencode CLI with the model argument, ensuring it's not prefixed with /v1
exec opencode "$@"
```

### Summary of Patches

1. **claude.sh**
    - Ensure `ANTHROPIC_BASE_URL` and `AUTH_TOKEN` are exported.
    - Pass the model argument directly to the `claude` command.

2. **aider.sh**
    - Ensure `OLLAMA_API_BASE` is exported.
    - Prefix the model name with `ollama/`.

3. **opencode.sh**
    - Ensure `url` is exported.
    - Pass the model argument directly without adding `/v1`.

### Example Usage

For a model like `qwen2.5-coder:14b`, you would run:

```sh
./claude.sh qwen2.5-coder:14b
./aider.sh qwen2.5-coder:14b
./opencode.sh qwen2.5-coder:14b
```

These patches ensure that each launcher correctly sets the necessary environment variables, handles model names appropriately, and maintains the model argument throughout execution chains.
