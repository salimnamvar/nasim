To address the issues with the agent launchers in `lib/nasim/agents/{claude.sh,aider.sh,opencode.sh}` and ensure that they can successfully use a real model like `qwen2.5-coder:14b` or `deepseek-r1:14b` over a forwarded URL, we need to carefully review each launcher script for potential bugs related to environment variable names, model name mangling, execution fallback chains, and missing exports.

### 1. **claude.sh**

**Issue:** 
- The script might not correctly handle the Ollama model name with the `ollama/` prefix.
- Missing export of necessary variables like `ANTHROPIC_BASE_URL` and `AUTH_TOKEN`.

**Patch:**
```sh
#!/bin/bash

# Ensure the correct environment variables are set
export ANTHROPIC_BASE_URL="http://localhost:8080"  # Update with your forwarded URL
export AUTH_TOKEN="ollama"

# Launch the Claude agent with the correct model name
exec claude-agent --model "ollama/qwen2.5-coder:14b"
```
